# 贡献指南

感谢你关注 **机电类毕业设计模拟答辩评委 Skill** 项目！本项目的目标是帮助机电类（机械/电气/自动化/机电一体化）本科毕业生在正式答辩前通过 AI 模拟评审发现问题、提升答辩质量。

欢迎任何形式的贡献——报告问题、提出改进建议、提交新内容或修复错误。

---

## 如何报告问题

如果你在使用过程中遇到问题或有改进建议，请通过 GitHub Issues 提交。提交时请包含以下信息：

1. **问题描述** — 清晰描述你遇到的问题或改进建议
2. **复现步骤**（如适用）— 详细说明如何复现该问题
3. **预期行为** — 描述你期望发生的事情
4. **实际行为** — 描述实际发生的事情
5. **环境信息** — 使用的 AI 工具名称/版本、加载的 prompt 方式等

---

## 如何提交改进

我们欢迎通过 Pull Request（PR）提交改进。请遵循以下流程：

1. **Fork 本仓库** — 点击 GitHub 页面右上角的 Fork 按钮
2. **创建分支** — 在 Fork 后的仓库中，从 `main` 分支创建新分支：
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **提交更改** — 在你的分支上进行修改，并提交清晰的 commit：
   ```bash
   git commit -m "feat: 简要描述你的更改内容"
   ```
4. **推送到远程** — 将分支推送到你 Fork 的仓库：
   ```bash
   git push origin feature/your-feature-name
   ```
5. **创建 Pull Request** — 在本仓库创建 Pull Request，目标分支为 `main`

### PR 提交清单

- [ ] 分支名称清晰反映改动目的（如 `fix/typo`, `feat/add-new-checklist`）
- [ ] Commit 信息遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范
- [ ] 改动内容与项目定位一致（仅涉及评审方法论相关内容）
- [ ] 如新增 checklist 或案例，确保内容为虚构/去标识化，不包含真实个人信息

---

## 代码 / 文档风格要求

- **语言**：中文为主，专业术语可保留英文（如 PID、PLC、CAD）
- **格式**：Markdown 格式，保持一致的标题层级和列表缩进
- **命名**：文件命名使用小写字母与连词符（如 `checklists/01-format.md`）
- **内容**：保持客观中立的评审语言，避免主观评价
- **示例**：案例内容必须是虚构的，不得包含任何真实课题、人名或评价记录

---

## 审查流程

1. 提交 PR 后，维护者会在 **3-5 个工作日** 内进行初审
2. 审查重点：内容准确性、格式合规性、隐私安全性（是否包含真实信息）
3. 可能需要修改或补充，我们会通过 PR 评论与你沟通
4. 审查通过后，将由维护者合并到 `main` 分支

---

## 行为准则

本项目采用 [贡献者公约（Contributor Covenant）](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) 作为行为准则。我们承诺为所有参与者营造一个开放、友善、尊重、无骚扰的社区环境。

我们**不欢迎**以下行为：
- 提交包含真实个人信息、真实课题内容或真实评价记录的内容
- 在任何渠道发布与本项目无关的商业广告或垃圾信息
- 对其他贡献者进行人身攻击、侮辱或歧视性言论

如有任何违反行为准则的情况，请通过 GitHub Issues 联系维护者处理。
