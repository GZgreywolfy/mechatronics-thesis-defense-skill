# 机电类毕业设计模拟答辩评委 Skill

一个用于 AI 的提示词 skill，让大模型扮演严格的机电类（机械 / 电气 / 自动化 / 机电一体化）本科毕业设计答辩评委，对毕业论文、图纸、PPT 或答辩自述进行模拟评审，帮助学生提前发现问题、补齐短板，从容应对正式答辩。

## 🎯 这个 Skill 能做什么

- **按五大维度审查材料**：格式规范、逻辑结构、技术深度、工程图纸、答辩表达
- **模拟评委现场追问**：针对技术薄弱点生成 2~5 个尖锐但合理的追问，提前预演答辩
- **给出可执行的修改建议**：每个问题附带「问题所在 → 为什么是问题 → 应该怎么改」
- **明确风险提示**：识别可能导致延期或二次答辩的核心风险（查重率、AI 代写疑云、实验数据缺失等）

## ⚡ 一分钟快速开始

### 第一步：将 Skill 设定加载到 AI 工具

打开 `SKILL.md` 文件，将其中的**角色设定（System Prompt）** 部分作为系统提示词注入你使用的 AI 工具。以下是各主流工具的配置方式：

#### ChatGPT（Web / App）

| 操作 | 说明 |
|------|------|
| **ChatGPT 网页版** | 点击左下角头像 → **Customize ChatGPT** → 在 **What would you like ChatGPT to know about you?** 下方的文本框中粘贴 `SKILL.md` 中从「角色设定(System Prompt)」到「红线提示」之间的全部内容 |
| **ChatGPT 临时配置** | 在对话开始时发送一条指令，例如：*"请你扮演一位机电类毕业设计答辩评委，以下是你需要遵循的评审规则：[粘贴 SKILL.md 内容]"* |

#### Claude（Web / API）

| 操作 | 说明 |
|------|------|
| **Claude 网页版** | 对话框下方点击 **Add Profile**（个人资料图标）→ 创建新 Profile → 在 **Profile Instructions** 中粘贴 `SKILL.md` 中从「角色设定(System Prompt)」到「红线提示」之间的全部内容 → 在需要使用时选择该 Profile 开启新对话 |
| **Claude API 用户** | 将 `SKILL.md` 角色设定内容放入 `system` 参数中 |

#### DeepSeek / 其他国产大模型

| 操作 | 说明 |
|------|------|
| **DeepSeek 网页版** | 大多数支持自定义"系统提示词"或"人设"功能，在设置中粘贴 `SKILL.md` 中从「角色设定(System Prompt)」到「红线提示」之间的全部内容即可 |
| **通用方法** | 如工具不支持预设系统提示词，可在对话开头发送：*"请你扮演一位严格的机电类本科毕业设计答辩评委，严格按以下规则评审：[粘贴 SKILL.md 内容]"* |

### 第二步：提交你的材料

加载角色设定后，直接提交你的论文章节、图纸描述、PPT 大纲或答辩自述。示例：

> "请用答辩评委 skill 审查我下面这一章——[粘贴你的绪论或技术章节内容]"
>
> "扮演评委，针对我这个'基于 STM32 的智能温室监测系统'提 5 个可能被追问的问题"
>
> "帮我检查我的机械装配图描述是否符合工程图纸规范"

## 🔧 机电类各专业方向适配说明

本 Skill 覆盖机电类四大专业方向，评审时对不同方向的侧重点有所差异。建议在使用时将你的**具体专业方向**告知 AI，以获得更有针对性的评审结果。

### ⚙️ 机械设计制造及其自动化

| 评审重点 | 典型关注点 |
|----------|-----------|
| **工程图纸** | 装配图/零件图数量与幅面、标注规范（GB/T）、公差配合、表面粗糙度、焊接符号 |
| **力学计算** | 关键零部件的强度/刚度/疲劳计算、公式代入正确性、量纲检查 |
| **结构设计** | 传动方案比较与选型依据、结构合理性、加工工艺可行性 |
| **实验验证** | 样机测试数据（转速/扭矩/效率/精度）、优化前后对比 |

> **推荐配套检查清单**：`checklists/04-engineering-drawing.md` 是本方向最关键的评审维度。

### ⚡ 电气工程及其自动化

| 评审重点 | 典型关注点 |
|----------|-----------|
| **电气原理** | 电路原理图规范性、电气元件选型依据（电流/电压/功率裕量） |
| **控制逻辑** | 控制策略（PID/PLC 编程/保护逻辑）的正确性与完整性 |
| **系统集成** | 供电方案、电磁兼容、接线图与端子排图的完整性 |
| **实验验证** | 波形/响应曲线/稳态误差/保护动作值的实测数据 |

> **提示**：提交材料时附带**电气原理图**和**关键波形图**的清晰描述，能使评审深度大幅提升。

### 🤖 自动化

| 评审重点 | 典型关注点 |
|----------|-----------|
| **建模与算法** | 系统建模方法（机理/辨识）、控制器设计（PID/模糊/自适应等）、算法实现 |
| **传感器与执行器** | 选型依据（量程/精度/响应时间）、信号调理电路设计 |
| **程序与系统** | 软件流程图/状态机、通信协议（Modbus/CAN/RS485）、人机界面设计 |
| **实验验证** | 阶跃响应/跟踪精度/抗干扰能力的定量对比数据 |

> **提示**：自动化方向的评委尤其关注**"你到底控了什么、控得多好"**——务必准备定量指标对比。

### 🔄 机电一体化

| 评审重点 | 典型关注点 |
|----------|-----------|
| **机电耦合** | 机械结构 → 传感器 → 控制器 → 执行器的完整信号/能量传递链路 |
| **接口匹配** | 机械接口与电气接口的对齐（如电机与负载的转矩-转速匹配） |
| **系统联调** | 机-电-控联调的实验数据、各子系统间的协调性 |
| **综合工作量** | 机械设计与电气/控制设计的工作量分布是否均衡 |

> **提示**：机电一体化是交叉学科，评委最关心的是"机械是否只是壳、控制是否只是板子"——必须体现机电融合的设计逻辑。

## 📋 完整使用流程

```
1. 加载 Skill 设定 ──→ 2. 选择专业方向 ──→ 3. 提交材料 ──→ 4. 获取评审 ──→ 5. 按建议修改
                              ↓
                     （可选）指明侧重维度
```

**建议提交流程**（按答辩前的时间线）：

| 时间节点 | 提交内容 | 目的 |
|----------|---------|------|
| 答辩前 2 周 | 完整论文初稿 | 全面审查五大维度 |
| 答辩前 1 周 | 修改后的论文 + 图纸 | 针对问题点复查 |
| 答辩前 3 天 | PPT 大纲 + 自述稿 | 演练答辩表达与时间把控 |
| 答辩前 1 天 | 完整模拟答辩 | 全流程预演，包括评委追问 |

## ❓ 常见问题 FAQ

### Q1：这个 Skill 的评审结果能替代导师意见吗？

**不能。** 本 Skill 的评审是模拟性质的辅助工具，不代表真实答辩评委的判断，更不能替代导师的专业指导。请务必将 AI 指出的问题与导师讨论后再做修改。

### Q2：评审结果会不会「误判」？

有可能。AI 评审基于通用规则，可能对特定课题的特殊要求（如学院的特殊格式规定、课题独有的技术路线）不够敏感。建议将 AI 评审当作"第二双眼睛"——它擅长发现常规问题，但最终判断仍需你结合实际情况。

### Q3：提交论文内容会泄露我的个人信息吗？

**不会。** 本 Skill 是纯本地提示词方案——你只需要将系统提示词粘贴到 AI 工具中，你的论文内容仅在当前对话中处理。但请注意：如果你使用 ChatGPT 等云端服务，建议不要在材料中包含真实姓名、学号等个人身份信息。详见下方的[数据来源与隐私声明](#-数据来源与隐私声明)。

### Q4：我该提交什么材料给 AI 评委？

任何与毕业设计相关的内容都可以：论文章节（建议分段提交，每次聚焦 1~2 章）、设计图纸的截图或详细描述、答辩 PPT 的大纲或逐页内容、答辩自述稿。如果是图纸评审，请尽可能详细描述图幅、标注内容、件号体系等。

### Q5：五大评审维度必须全部审查吗？

不一定。你可以指定只审查某几个维度，例如只想检查"技术深度"和"工程图纸"。在输入时明确告知 AI 即可，如："请重点审查技术内容深度与工作量维度，其他维度略过。"

### Q6：多个方向的审查标准有什么不同？

参见上方的「[机电类各专业方向适配说明](#-机电类各专业方向适配说明)」章节。不同方向的评审侧重点差异较大，建议在使用时明确告知 AI 你的专业方向。

### Q7：为什么 AI 有时候看不出图纸的问题？

AI 模型（尤其是纯文本模型）无法直接读取图纸中的图形元素。图纸评审的效果取决于你对图纸的文字描述是否详细。建议：描述图幅大小、标注内容、件号、标题栏信息等；如果有条件，使用支持图像识别的多模态模型（如 GPT-4V、Claude Vision）上传图纸截图。

### Q8：这个 Skill 对硕士研究生论文也有用吗？

本 Skill 的评审标准和检查点主要针对**本科毕业设计（论文）** 设计。硕士论文在创新性、理论深度、实验系统性和论文篇幅上要求更高，直接使用本 Skill 可能会遗漏一些高级审查点。建议硕士生将其作为基础评审工具，再结合学科特点做补充。

## 🔒 数据来源与隐私声明

本 Skill 是**基于真实毕业设计答辩场景提炼出的通用评审方法论**，所有内容已彻底去标识化，**不包含任何真实个人信息、真实课题或真实评价记录**。`examples/` 中的案例均为虚构。

> ⚠️ **使用提醒**：虽然本 Skill 本身不含个人数据，但你在使用 AI 工具提交论文内容时，**建议自行删除或模糊化处理姓名、学号、导师姓名、学校名称等个人身份信息**，以保护隐私。
