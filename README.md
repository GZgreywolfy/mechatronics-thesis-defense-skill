# 机电类毕业设计模拟答辩评委 Skill

## 🎯 这个 Skill 能做什么

- **按五大维度审查材料**：格式规范、逻辑结构、技术深度、工程图纸、答辩表达
- **模拟评委现场追问**：针对技术薄弱点生成 2~5 个尖锐但合理的追问，提前预演答辩
- **给出可执行的修改建议**：每个问题附带「问题所在 → 为什么是问题 → 应该怎么改」
- **明确风险提示**：识别可能导致延期或二次答辩的核心风险（查重率、AI 代写疑云、实验数据缺失等）

## 📖 核心提示词预览

以下是 `SKILL.md` 中角色设定（System Prompt）的核心片段，让你无需下载即可感受本 Skill 的评审风格：

> 你是一位机电类专业的资深毕业设计答辩评委，熟悉本科毕业设计（论文）的规范要求、工程图纸标准与答辩流程。你的评审风格直接、专业、抓重点，既指出问题也给出可执行的修改方向。
>
> 当用户提交毕业设计材料时，你需要：
>
> 1. **按五大评审维度逐项审查**：论文格式与规范、论文逻辑与结构、技术内容深度与工作量、工程图纸规范、答辩表达与演示
> 2. **对每个问题**，指出"问题所在 → 为什么是问题 → 应该怎么改"
> 3. **模拟评委追问**：针对技术内容提出 2~5 个尖锐但合理的问题
> 4. **给出总体结论与风险提示**
>
> **红线提示**：查重率可能超过学院红线、内容疑似 AI 代写且学生无法解释、声称完成实物但无实验结果支撑——这些问题必须主动指出。

完整角色设定请查看 [SKILL.md](SKILL.md) 文件。

## ⚡ 一分钟快速开始

### 第一步：将 Skill 设定加载到 AI 工具

打开 `SKILL.md` 文件，将其中的**角色设定（System Prompt）** 部分作为系统提示词注入你使用的 AI 工具。以下是各主流工具的配置方式：

#### ChatGPT（Web / App）

| 操作 | 说明 |
|------|------|
| **ChatGPT 网页版** | 点击左下角头像 → **Customize ChatGPT** → 在 **What would you like ChatGPT to know about you?** 下方的文本框中粘贴 `SKILL.md` 中从「角色设定(System Prompt)」到「红线提示」之间的全部内容 |
| **ChatGPT 临时配置** | 在对话开始时发送一条指令，例如：*"请你扮演一位机电类毕业设计答辩评委，以下是你需要遵循的评审规则：[粘贴 SKILL.md 内容]"* |

#### Claude（Web / API）

| 操作 | 说明 |
|------|------|
| **Claude 网页版** | 对话框下方点击 **Add Profile**（个人资料图标）→ 创建新 Profile → 在 **Profile Instructions** 中粘贴 `SKILL.md` 中从「角色设定(System Prompt)」到「红线提示」之间的全部内容 → 在需要使用时选择该 Profile 开启新对话 |
| **Claude API 用户** | 将 `SKILL.md` 角色设定内容放入 `system` 参数中 |

#### DeepSeek / 其他国产大模型

| 操作 | 说明 |
|------|------|
| **DeepSeek 网页版** | 大多数支持自定义"系统提示词"或"人设"功能，在设置中粘贴 `SKILL.md` 中从「角色设定(System Prompt)」到「红线提示」之间的全部内容即可 |
| **通用方法** | 如工具不支持预设系统提示词，可在对话开头发送：*"请你扮演一位严格的机电类本科毕业设计答辩评委，严格按以下规则评审：[粘贴 SKILL.md 内容]"* |

### 第二步：提交你的材料

加载角色设定后，直接提交你的论文章节、图纸描述、PPT 大纲或答辩自述。示例：

> "请用答辩评委 skill 审查我下面这一章——[粘贴你的绪论或技术章节内容]"
>
> "扮演评委，针对我这个'基于 STM32 的智能温室监测系统'提 5 个可能被追问的问题"
>
> "帮我检查我的机械装配图描述是否符合工程图纸规范"

## 🎬 一分钟体验示例

以下展示一段完整的模拟答辩评审对话，让你快速感受本 Skill 的实际效果。

### 学生提交

> **课题**：基于 STM32 的智能温室环境监测系统设计
>
> 摘要要点：系统以 STM32F103 为主控，采集温湿度与 CO₂ 浓度，通过 OLED 显示并经蓝牙上传至手机端。声称完成了实物并进行了"优化"，效率较传统方案提升了 20%。

### AI 评委反馈（节选）

**【技术内容深度与工作量】**
- **问题**：声称"优化后效率提升 20%"，但未给出优化前后量化对比数据，也未说明设计目标值。
- **为什么是问题**：无对比的"优化"属于主观断言，无法判断真实效果，评委有理由质疑数据真实性。
- **修改建议**：补充优化前/后关键指标对照表，标明设计目标值与实际达成值。

**【模拟评委追问】**
1. "你这个系统的设计目标指标具体是什么？温湿度精度要求是多少？"
2. "你说完成了实物，实测数据在哪一章？做了多少次测试？"
3. "你所说的优化，优化前和优化后分别是什么数值？请拿数据说话。"

**【总体结论】**
- 🔴 **高风险**：缺少实验数据支撑，"优化提升"无对比，可能导致答辩不通过
- 🟡 **中风险**：系统方案缺乏选型论证，STm32 选型理由不充分
- 修改优先级：补实验数据 > 重写选型论证 > 调整格式

> 更多完整示例请参阅 [examples/](examples/) 目录下的案例文件。

## 🔧 机电类各专业方向适配说明

本 Skill 覆盖机电类四大专业方向，评审时对不同方向的侧重点有所差异。建议在使用时将你的**具体专业方向**告知 AI，以获得更有针对性的评审结果。

### ⚙️ 机械设计制造及其自动化

| 评审重点 | 典型关注点 |
|----------|-----------|
| **工程图纸** | 装配图/零件图数量与幅面、标注规范（GB/T）、公差配合、表面粗糙度、焊接符号 |
| **力学计算** | 关键零部件的强度/刚度/疲劳计算、公式代入正确性、量纲检查 |
| **结构设计** | 传动方案比较与选型依据、结构合理性、加工工艺可行性 |
| **实验验证** | 样机测试数据（转速/扭矩/效率/精度）、优化前后对比 |

> **推荐配套检查清单**：`checklists/04-engineering-drawing.md` 是本方向最关键的评审维度。

### ⚡ 电气工程及其自动化

| 评审重点 | 典型关注点 |
|----------|-----------|
| **电气原理** | 电路原理图规范性、电气元件选型依据（电流/电压/功率裕量） |
| **控制逻辑** | 控制策略（PID/PLC 编程/保护逻辑）的正确性与完整性 |
| **系统集成** | 供电方案、电磁兼容、接线图与端子排图的完整性 |
| **实验验证** | 波形/响应曲线/稳态误差/保护动作值的实测数据 |

> **提示**：提交材料时附带**电气原理图**和**关键波形图**的清晰描述，能使评审深度大幅提升。

### 🤖 自动化

| 评审重点 | 典型关注点 |
|----------|-----------|
| **建模与算法** | 系统建模方法（机理/辨识）、控制器设计（PID/模糊/自适应等）、算法实现 |
| **传感器与执行器** | 选型依据（量程/精度/响应时间）、信号调理电路设计 |
| **程序与系统** | 软件流程图/状态机、通信协议（Modbus/CAN/RS485）、人机界面设计 |
| **实验验证** | 阶跃响应/跟踪精度/抗干扰能力的定量对比数据 |

> **提示**：自动化方向的评委尤其关注**"你到底控了什么、控得多好"**——务必准备定量指标对比。

### 🔄 机电一体化

| 评审重点 | 典型关注点 |
|----------|-----------|
| **机电耦合** | 机械结构 → 传感器 → 控制器 → 执行器的完整信号/能量传递链路 |
| **接口匹配** | 机械接口与电气接口的对齐（如电机与负载的转矩-转速匹配） |
| **系统联调** | 机-电-控联调的实验数据、各子系统间的协调性 |
| **综合工作量** | 机械设计与电气/控制设计的工作量分布是否均衡 |

> **提示**：机电一体化是交叉学科，评委最关心的是"机械是否只是壳、控制是否只是板子"——必须体现机电融合的设计逻辑。

## ❓ 常见问题 FAQ

### Q1：这个 Skill 的评审结果能替代导师意见吗？

**不能。** 本 Skill 的评审是模拟性质的辅助工具，不代表真实答辩评委的判断，更不能替代导师的专业指导。请务必将 AI 指出的问题与导师讨论后再做修改。

### Q2：评审结果会不会「误判」？

有可能。AI 评审基于通用规则，可能对特定课题的特殊要求（如学院的特殊格式规定、课题独有的技术路线）不够敏感。建议将 AI 评审当作"第二双眼睛"——它擅长发现常规问题，但最终判断仍需你结合实际情况。

### Q3：提交论文内容会泄露我的个人信息吗？

**不会。** 本 Skill 是纯本地提示词方案——你只需要将系统提示词粘贴到 AI 工具中，你的论文内容仅在当前对话中处理。但请注意：如果你使用 ChatGPT 等云端服务，建议不要在材料中包含真实姓名、学号等个人身份信息。详见下方的[数据来源与隐私声明](#-数据来源与隐私声明)。

### Q4：我该提交什么材料给 AI 评委？

任何与毕业设计相关的内容都可以：论文章节（建议分段提交，每次聚焦 1~2 章）、设计图纸的截图或详细描述、答辩 PPT 的大纲或逐页内容、答辩自述稿。如果是图纸评审，请尽可能详细描述图幅、标注内容、件号体系等。

### Q5：五大评审维度必须全部审查吗？

不一定。你可以指定只审查某几个维度，例如只想检查"技术深度"和"工程图纸"。在输入时明确告知 AI 即可，如："请重点审查技术内容深度与工作量维度，其他维度略过。"

### Q6：多个方向的审查标准有什么不同？

参见上方的「[机电类各专业方向适配说明](#-机电类各专业方向适配说明)」章节。不同方向的评审侧重点差异较大，建议在使用时明确告知 AI 你的专业方向。

### Q7：为什么 AI 有时候看不出图纸的问题？

AI 模型（尤其是纯文本模型）无法直接读取图纸中的图形元素。图纸评审的效果取决于你对图纸的文字描述是否详细。建议：描述图幅大小、标注内容、件号、标题栏信息等；如果有条件，使用支持图像识别的多模态模型（如 GPT-4V、Claude Vision）上传图纸截图。

### Q8：这个 Skill 对硕士研究生论文也有用吗？

本 Skill 的评审标准和检查点主要针对**本科毕业设计（论文）** 设计。硕士论文在创新性、理论深度、实验系统性和论文篇幅上要求更高，直接使用本 Skill 可能会遗漏一些高级审查点。建议硕士生将其作为基础评审工具，再结合学科特点做补充。

## 🖼️ 配合多模态模型使用

本 Skill 的核心评审逻辑基于文本分析，但如果你使用的 AI 工具支持图像识别（如 GPT-4V、Claude Vision、Gemini 等多模态模型），你可以直接上传图纸截图、PPT 页面图片或论文中的图表，让评委进行更直观的审查。

### 支持多模态的 AI 工具配置

| AI 工具 | 多模态支持情况 | 配置说明 |
|---------|---------------|---------|
| **ChatGPT（GPT-4V）** | ✅ 支持上传图片 | 加载 Skill 设定后，直接在对话框中点击"+"按钮上传图片，然后输入审查需求，例如："请审查这张装配图的标注规范性" |
| **Claude（Claude Vision）** | ✅ 支持上传图片 | 在对话中点击回形针图标上传图片文件（支持 PNG、JPG、JPEG、GIF、WebP），然后说明审查需求 |
| **Gemini** | ✅ 支持上传图片 | 支持上传图片并进行分析，操作方式类似 |
| **DeepSeek** | ⚠️ 有限支持 | 部分版本支持图片上传，功能在持续更新中，具体请查看官方文档 |

### 上传图纸截图的最佳实践

为获得最佳的评审效果，上传图纸截图时请遵循以下建议：

#### 1. 截图质量要求

- **分辨率**：截图分辨率不低于 1920×1080 像素，确保尺寸标注线和数字清晰可辨
- **文件格式**：优先使用 PNG（无损格式），避免 JPEG 压缩导致文字模糊
- **图纸内容**：截取完整视图，不要只截取局部——评委需要看到图框、标题栏、标注、技术要求等全部信息
- **多张图纸**：如果有多张图纸，逐张上传，每张附上对应的审查需求

#### 2. 图纸描述模板

即使上传了截图，建议同时提供简要的文字描述，帮助 AI 更准确地理解图纸内容。推荐使用以下模板：

```markdown
【图纸名称】[减速器装配图 / 箱体零件图 / 电气原理图]
【图幅】[A0 / A1 / A2 / A3]
【主要视图】[主视图 / 剖视图 / 局部放大图]
【关键标注】[配合尺寸 Ø62H7 / 形位公差 / 表面粗糙度 Ra 值]
【审查重点】[请帮我检查尺寸标注完整性 / 公差配合合理性 / 视图表达充分性]
```

#### 3. 常见图纸类型的最佳上传方式

| 图纸类型 | 建议处理方式 | 说明 |
|---------|------------|------|
| **机械装配图** | 截图后上传主视图 + 主要剖视图 | 分两次上传，先传整体视图审查标注规范，再传局部放大图审查配合关系 |
| **零件图** | 上传完整零件图 | 确保尺寸标注、公差、粗糙度符号均在截图中清晰可见 |
| **电气原理图** | 截取完整电路图，分段上传 | 电路图通常幅面较大，建议按功能模块分块上传（如"电源模块""主控模块""驱动模块"） |
| **PCB 布局图** | 上传顶层和底层截图 | 确保元器件标号和走线可见 |
| **控制流程图** | 上传完整流程图 | 确保判断条件和路径标注清晰 |
| **仿真结果图** | 上传含图例和坐标轴标注的完整截图 | 务必包含横纵坐标的物理量名称和单位 |

#### 4. 注意事项

- **不要过度压缩**：保留截图的原始分辨率，避免缩小后关键标注模糊
- **标注关键区域**：如有特定审查点，可在截图上用红圈或箭头标注后上传
- **配套文字说明**：上传图片后，请用文字补充说明你的审查需求，例如："请重点检查这张装配图中轴承配合处的公差标注是否完整"
- **注意隐私**：上传前检查图纸标题栏中是否包含个人信息（姓名、学号等），如有可打码处理后上传

## 📋 常见错误速查

以下是各 checklists 中机电类毕业设计最常见错误的分类汇总，供你在准备阶段快速对照自查。

### 维度一：论文格式与规范

| 常见错误 | 严重程度 | 简要说明 |
|---------|---------|---------|
| 参考文献格式不统一 | ⚠️ 高频 | 同一论文中混用多种文献格式，或缺少文献类型标识（[J]、[D]等） |
| 章节编号跳级或混乱 | 🔴 硬伤 | 编号不连续（如 2.1 → 2.3），或使用四级标题（超过本科论文要求） |
| 字体混用 | ⚠️ 常见 | 正文中宋体、楷体、仿宋混用，英文未使用 Times New Roman |
| 未使用学院模板 | 🔴 硬伤 | 套用非本院模板，页眉页脚信息错误，或未按模板修改个人信息 |
| 图号/表号与交叉引用不一致 | ⚠️ 常见 | 文中称"如图 3-2"，实际编号为"图 3-1"；表格编号跳号 |
| 目录与正文页码不对应 | ⚠️ 常见 | 目录页未更新，页码与实际正文内容不符 |
| 单位与变量符号不规范 | ⚠️ 常见 | 单位大小写混淆（KG/kg、MM/mm），变量符号未用斜体 |

### 维度二：论文逻辑与结构

| 常见错误 | 严重程度 | 简要说明 |
|---------|---------|---------|
| 背景与内容脱节 | 🔴 硬伤 | 绪论大篇幅讨论行业趋势，正文做的是具体控制系统，二者无衔接 |
| 参数凭空出现 | ⚠️ 常见 | 计算中关键参数（如 F=500N）直接给出，无选型依据或推导过程 |
| 章节顺序不合理 | ⚠️ 常见 | 分析放在设计之前（仿真时读者还未知设计对象） |
| 小结形同虚设 | ⚠️ 常见 | 没有小结，或写成了"本章首先……然后……最后……"的流水账 |
| 方案无对比 | 🔴 硬伤 | 直接给出"本方案采用 XXXX"，通篇未见多方案对比和选型依据 |
| 论文拼凑感强 | 🔴 硬伤 | 多个独立项目拼接在一起，章节之间无逻辑联系 |

### 维度三：技术内容深度与工作量

| 常见错误 | 严重程度 | 简要说明 |
|---------|---------|---------|
| 声称"优化"但无对比数据 | 🔴 硬伤 | 仅展示两种方案，未给出量化改进幅度 |
| 实验数据缺失 | 🔴 硬伤 | 声称完成了实物/测试，但无任何具体数据支撑 |
| 公式代入错误 | 🔴 硬伤 | 公式写 A/B，代入变成 A×B；或代入时未做单位换算 |
| 选型依据不足 | ⚠️ 常见 | 只写芯片/器件参数，不写选型理由 |
| 通用知识充篇幅 | ⚠️ 常见 | 大面积抄写教材公式或芯片手册作为论文内容 |
| 成品当自研 | 🔴 硬伤 | 购买成品模块未声明，论文表述模糊为"本设计采用" |
| 测试方法不科学 | ⚠️ 常见 | 单次测试、无控制变量、无误差分析 |
| 目标与结果脱节 | ⚠️ 常见 | 设计目标和实验验证分属不同维度，论文逻辑断裂 |

### 维度四：工程图纸规范

| 常见错误 | 严重程度 | 简要说明 |
|---------|---------|---------|
| 重复图纸凑数 | ⚠️ 常见 | 对称件出两张图、相同结构重复出图 |
| 标准件出零件图 | ⚠️ 常见 | 电机/减速器/轴承等外购件单独出零件图 |
| 件号体系不一致 | ⚠️ 常见 | 装配图件号与零件图编号不对应 |
| 中心线与轴线混淆 | ⚠️ 常见 | 线型不分、连在一起不断开 |
| 封闭尺寸链 | 🔴 硬伤 | 各段尺寸加和等于总长且全部标注 |
| 装配标注无说明 | ⚠️ 常见 | A/B/C 分组选配含义未在图纸中解释 |
| 电气符号错误 | 🔴 硬伤 | 线圈/触点符号混用、方向画反 |
| 形位公差基准错误 | ⚠️ 常见 | 基准选在非加工面或不合理位置 |
| 剖视图剖面线不一致 | ⚠️ 常见 | 同一零件不同视图剖面线方向/间距不同 |
| 主视图选择不当 | ⚠️ 常见 | 未选择信息量最大的方向作为主视图 |

### 维度五：答辩表达与演示

| 常见错误 | 严重程度 | 简要说明 |
|---------|---------|---------|
| PPT 照搬论文 | 🔴 硬伤 | PPT 大段粘贴论文原文，无提炼、无排版 |
| 时间把控失当 | ⚠️ 常见 | 背景部分占用超过 50% 时间，核心工作草草带过 |
| 演示准备不足 | 🔴 硬伤 | 实物演示出现故障且无备用方案，无法展示核心功能 |
| 对 AI 辅助内容无法解释 | 🔴 硬伤 | 被追问时表示"这段是 AI 写的，我不太清楚" |
| 过度夸大工作量 | ⚠️ 常见 | 将团队/学期工作量说成个人独立完成，被评委识破 |
| 流程图/代码截图字号过小 | ⚠️ 常见 | 答辩时后排评委完全看不清 |
| 数据展示单薄 | ⚠️ 常见 | 只有成功率数字，无统计分析和多组对照 |
| 章节标题页过多 | ⚠️ 常见 | 纯标题页占 1/3 页数，浪费有效展示时间 |

> **使用建议**：将上表与各 checklists 中的详细错误案例配合使用。每个维度的高频错误和硬伤错误应优先排查。
